# Curriculum evidence matrix

This matrix is the durable course-level conformance record. A row becomes
`covered` only after its learner-facing explanation, runnable demonstration,
practice feedback, and any required independent application have been
verified. Topic names and incidental use do not count as evidence.

The final reconciliation uses `course-manifest.json`, runnable lesson artifacts,
solution-selected MTP tests, starter-feedback checks, Tasks and capstone
acceptance suites, independent coverage gates, and the root Release workflow. Exact validation
results are recorded in [CONFORMANCE.md](CONFORMANCE.md).

## Cross-course area reconciliation

These area rows complement the instructional-unit evidence above. They prevent a
topic mention in one place from substituting for the connected teaching path.

| Area | Status | Target evidence and scope rationale |
| --- | --- | --- |
| Program execution, syntax, and control flow | covered | Lessons 01-03 name, explain, demonstrate, practice, and apply the execution and control-flow model. |
| Scalar values, nullability, conversions, and Boolean conditions | covered | Lessons 02-03 plus nullable analysis and downstream validation. C# requires explicit Boolean conditions rather than general truthiness. |
| Numeric representation, precision, and overflow | covered | Lesson 02 connects integer division, `decimal`, binary floating point, `NaN`, and checked integral overflow to observable behavior and feedback. |
| Unicode text, bytes, raw/structured data, and I/O | covered | Lessons 02, 11, and 14 connect object -> JSON text -> UTF-8 bytes -> stream/file/HTTP through explanation, runnable behavior, practice, and applied use. |
| Collections, mappings, sets, iteration, mutation, copying, and equality | covered | Lessons 04, 07, and 09 distinguish container choice, aliases, shallow container copies, identity, value equality, and element equality before capstone use. |
| Methods, invocation, parameter passing/binding, and returns | covered | Lesson 05 explains pass-by-value, reference-type mutation versus reassignment, return values, overloads, and the later `out`/Try pattern. |
| Native object and data modeling | covered | Lessons 07-08 cover records, structs, classes, enums, composition, interfaces, generics, delegates, and invariant boundaries. |
| Errors, resource ownership, cleanup, and lifecycle | covered | Lessons 05 and 11-15 connect specific exceptions, synchronous/async disposal, transactions, atomic replacement, cancellation, HTTP responses, and host shutdown. |
| Files, directories, paths, streams, buffers, HTTP, persistence, and formats | covered | Lessons 11, 13, and 14 plus Tasks/capstones cover file, SQLite, Markdown, process, and network surfaces. |
| Dates, times, durations, time zones, timestamps, and clocks | covered | Lesson 07 teaches `DateOnly`, `DateTimeOffset`, UTC, `TimeSpan`, strict ISO parsing, and clock input; Lessons 12 and 14 practice durations; the idiomatic capstone applies dates/timestamps. Broader time-zone conversion is outside scope. |
| Projects, namespaces, packages, build/distribution, and public API documentation | covered | Lessons 06, 08, and 15 cover SDK projects, `.slnx`, portable lock-file boundaries, publish artifacts, XML documentation, and the non-package boundary. |
| Idiomatic repository structure and generated-artifact boundaries | covered | Shared `lessons/`, `exercises/`, `projects/`, and `capstones/` roles contain .NET-native projects/namespaces; applied destinations enforce one top-level type per matching file and inward project dependencies. |
| CLI, environment, OS/process interoperability, streams, and exit behavior | covered | Setup, Lesson 15, Tasks, and both capstones cover arguments, environment, stdout/stderr, exit codes, subprocesses, and cleanup across claimed OSes. |
| Static typing, analyzers, nullability, and native abstractions | covered | Shared build configuration and Lessons 02, 06-10 make compiler, nullable, analyzer, generic, and delegate feedback observable. |
| Testing, debugging, formatting, analysis, coverage, and CI | covered | Lessons 05, 10, and 15 plus destination-specific gates teach the widening lifecycle with actionable failure paths and local/CI parity. |
| Async, cancellation, concurrency, and owned lifecycle | covered | Lesson 12 provides explanation, runnable, exercise, and feedback; Lessons 14-15, Tasks, and the idiomatic capstone apply owned async I/O and cancellation. |
| Persistence/API validation and untrusted input | covered | Lessons 11, 13, and 14 plus Tasks/capstone tests validate paths, SQL parameters, stored data, DTOs, JSON, statuses, base URLs, and strict dates. |
| Beginner-appropriate algorithms and performance | covered | Lessons 04, 09, and 12 teach container selection, repeated work, materialization, and bounded concurrency. Formal algorithm analysis is an explicit non-goal. |
| Theory-to-runnable-to-exercise-to-project/capstone continuity | covered | The manifest, 15 sequenced lessons, exercise feedback, required Tasks bridge, and both capstone tracks establish the complete evidence progression. |
